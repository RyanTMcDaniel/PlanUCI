SUPABASE SETUP : Set up 13 different tables for course information. I am pulling the info from AnteaterAPI. I am seperating entities based on how often the information changes (think Course Description vs Course Building Number). Doing this allows for safer updating as their is less info to overwrite on each 'refresh'. Mirrored my tables by the sshape of the data returned by API JSON responses to make storing easier. Setup RLS in Supabase to block unauthorized access. This keeps user-specific data safe and only readable by allowed users. Backend import scripts use the service-key role which bypasses RLS, so it never goes in frontent code or the public repo, only in the server environment variables.

Pulled UCI Catalog : All 129 departments, 6670 courses, 3279 instructors. Decideed to pull everything rather than just STEM and GEs. Filtering would have led to gaps in courses and not have been as extensive as I'm wanting to be. Still haven't gone through to properly align prereqs and major requirements : I initially was hoping to pull major requirements from AnteaterAPI but they aren't stored there so I had to scrape each major's requirement page. Currently I have only done STEM+Business majors but I'm hoping to go back and include humanities. The way the courses are listed isn't able to be properly scraped for sections where you 'pick x# out y# courses' so I will have to manually input those to Supabase unfortunately.

Difficulty Scoring NLP Model : Fine tuned all-MiniLM-L6-v2 for 3-class classification. Classification boundaries were set using percentile splits (20/60/20) computed on training set only and then applied to val and test). Fixed boundaries initially produced class imbalances with only 10/1674 courses being labelled 'Hard'. The GPA clusters too tightly for a uniform 1-10 scale to work. Final test macro F1: 0.619 with accuracy of 65.9%. Per tier results (Easy F1: 0.695, Medium F1: 0.710, Hard F1: 0.452). Hard F1 being weakest was expected since description text alone cannot determine grading strictness, I will address this in next implementation where I consider GPA distribution and RateMyProfessor signals into overall difficulty rating. Started off with a regression model being trained on MSE and switched to classification model being trained with cross-entropy loss. Also had to oversample hard and easy tiers to fix class imbalances as 60% of data was pre-labeled as medium. Filtered out grad courses and participation-graded courses (Assumed to be courses with exactly 4.0GPA average).
